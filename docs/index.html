ggplot2 の基礎：The Layered Grammar of Graphics

ggplot2 は、R でデータの可視化を行うときに標準的に用いられているパッケージです。

ggplot(mtcars, aes(x = hp, y = mpg)) +
  geom_smooth(method = "gam", formula = y ~ s(x, bs = "cs"), 
              color = "grey80", fill = "grey90", alpha = 0.5) +
  geom_point(aes(size = wt, color = factor(cyl)), alpha = 0.8) +

    size = "Weight",
    color = "Cylinders"
  ) +
  theme_midr(grid = "xy") +
  theme(legend.position = "bottom")

Hadley Wickham は、2008年の論文の中で “A layered grammar of graphics” という考え方を提唱し、この考え方に基づいて ggplot2 を実装しました。 ggplot2 の関数群は、

プロットは、以下の構成要素から成り立っています：

  1. defaults：グラフで共通して使用する data と mapping を定める基盤。

  • data：データセット

  • mapping：幾何オブジェクトのどの美的属性 (位置、色、サイズなど) にどの変数を割り当てるかを定める規則

  1. layer：データの可視化を担うレイヤー。

　- data 　 　- mapping 　 　- geom 幾何オブジェクト (Geometric Object) (point, line, bar, …) 　 　- stat: 統計的変換 (identity, count, smoothなど)

　- position (Position adjustment)：位置の調整 (stack, dodge, jitterなど)

  1. scale: データから美的属性へのマッピング。

  2. coord (Coordinate System)：幾何オブジェクトの位置 (“x”, “y”, “xmin”, “xmax” などの値) をプロット上の領域に移す方法を定める規則。

  3. facet: データの分割 (パネル分け) 。

この「文法」を愚直にコードに落とし込むと、以下のように記述することになります。

ggplot() +
  layer(
    data = diamonds,
    mapping = aes(x = carat, y = price),
    geom = "point",
    stat = "identity",
    position = "identity"
  ) +
  scale_y_continuous() +
  scale_x_continuous() +
  coord_cartesian()
  • ggplot()：プロットを作成するための関数。defaults に対応していて、data と mapping を指定する。
  • aes()：変数と美的属性の対応付けを mapping として指定できるようにまとめるための関数。
  • layer()：レイヤーを作成するための関数。
  • scale_[x]_[continuous]()：美的属性ごとにスケールの種類を指定するための関数。
  • coord_[cartesian]()：座標系の種類を指定するための関数。

しかし、実際には標準的なレイヤーを作成するための便利なショートカット関数 (geom_point() など) が多数用意されており、スケールや座標系もデータに対して適切なものが自動的に選択されるため、私たちが普段書くコードは驚くほどシンプルになります。

ggplot(data = diamonds, mapping = aes(x = carat, y = price)) +
  geom_point()

データの構造を「色」にする：3つのカラーパレット

美的属性の中でも、色 (Color / Fill) 」は人間の視覚に最も強く訴えかける要素です。 しかし、データの構造に合わせて「正しいパレット」を選ばないと、グラフはかえって読みにくくなります。

カラースケールは、大きく以下の３種類に分類されます。 データの性質に合わせてこれらを使い分けるのが、美しいビジュアライゼーションの鉄則です。

質的パレット (Qualitative)

適したデータ: カテゴリ変数 (例：大陸、性別、血液型) 。

特徴: 色同士に明確な順序や大小関係が「ない」パレットです。各カテゴリがはっきりと区別できることが最優先されます。

par.midr(mfrow = c(3, 1))
plot(color.theme("okabeito"))
plot(color.theme("Austria"))
plot(color.theme("moon"))

順序・連続パレット (Sequential)

適したデータ: ゼロから一方向に増加する連続変数 (例：人口、降水量、価格) 。

特徴: 明度や彩度が、低い値から高い値に向かって一方向に滑らかに変化します。

par.midr(mfrow = c(3, 1))
plot(color.theme("viridis"))
plot(color.theme("mako"))
plot(color.theme("magma"))

発散パレット (Diverging)

適したデータ: 明確な「中央値」や「基準値」があり、そこからプラス・マイナスに広がる連続変数 (例：気温、基準年からの増減率) 。

特徴: 中央を中立な色 (白や薄いグレー) とし、両極端に向かって異なる2つの色相が濃くなっていきます。

par.midr(mfrow = c(3, 1))
plot(color.theme("RdBu"))
plot(color.theme("Hiroshige"))
plot(color.theme("eclipse"))
  1. まとめ {ggplot2} は、データを Aesthetics にマッピングする強力なキャンバスです。 幾何的オブジェクト (Geom) でデータの「骨格」を作り、データ構造に合致した「3種類のカラーパレット」を適用することで、プロットは単なる図表から、直感的で説得力のある「メッセージ」へと進化します。

ぜひ皆様も、{ggplot2} の拡張パッケージ (ggplot2 extensions) や独自のテーマパッケージを駆使して、ご自身のデータに最適な「色」を与えてみてください。
