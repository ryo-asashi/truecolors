The Layered Grammar of Graphics

  • プロットは、図形的要素やデータに対する統計処理などの抽象的な構成要素からなる Layer を重ねて構築されるもの

  1. Defaults: プロットで共通して用いる data と mapping

    mapping: 変数と審美的属性の対応関係 / aes() 関数で指定

  2. Layer: 図形的要素 geom / 統計的変換 stat / 配置調整 position

  3. Scale: データを位置や色などの審美的属性のスケール上の値に変換

  4. Coord: 位置の値を描画領域の物理的な場所に対応付ける座標系

  5. Facet: データを分割して小窓で表示する方法の指定

厳密なコード表現

ggplot() +
  layer(
    data = diamonds,
    mapping = aes(x = carat, y = price),
    geom = "point",
    stat = "identity",
    position = "identity"
  ) +
  scale_y_continuous() +
  scale_x_continuous() +
  coord_cartesian()

実際のコード

ggplot(diamonds, aes(carat, price)) +
  geom_point()

  • geom_*() や stat_*() は必要な要素をデフォルト値で補間した Layer を出力

  • scale_*() や coord_*() はデフォルトの設定が適用される

散布図 (Scatter Plot)

  • geom = "point", stat = "identity", position = "identity"

  • geom_point() で作成

ggplot(
  data = diamonds,
  mapping = aes(x = carat, y = price)
) +
  geom_point() +
  scale_x_log10() +
  scale_y_log10()

棒グラフ (Bar Chart) - 度数グラフ

  • geom = "bar", stat = "count", position = "stack"

  • geom_bar() で作成

ggplot(
  data = diamonds,
  mapping = aes(x = clarity)
) +
  geom_bar()

棒グラフ (Bar Chart) - 列グラフ

  • geom = "col", stat = "identity", position = "stack"

  • geom_col() で作成

ggplot(
  data = diamonds |> count(clarity),
  mapping = aes(x = clarity, y = n)
) +
  geom_col()

ヒストグラム (Histogram)

  • geom = "bar", stat = "bin", position = "stack"

  • geom_histogram() で作成

ggplot(
  data = diamonds,
  mapping = aes(x = price)
) +
  geom_histogram(bins = 20) +
  scale_x_log10()

密度曲線 (Density Plot)

  • geom = "area", stat = "density", position = "identity"

  • geom_density() で作成

ggplot(
  data = diamonds,
  mapping = aes(x = price)
) +
  geom_density() +
  scale_x_log10()

箱ひげ図 (Boxplot)

  • geom = "boxplot", stat = "boxplot", position = "dodge2"

  • geom_boxplot() で作成

  mapping = aes(x = clarity, y = price)

データの構造を色にする

  • 色は審美的属性の中でも特に人間の視覚に強く訴えかける

  • データの性質に即したスケールを選ばないと解釈を歪めてしまう

  • 3種類の基本パレット

    1. Qualitative (質的)

    2. Sequential (順序)

    3. Diverging (発散)

Qualitative: 質的カラーパレット

  • 順序や大小関係のない配色

  • 因子型変数の塗り分けなどに利用する

Okabe-Ito: カラーユニバーサルデザイン

HCL: 視覚的平等を保つ設計

Sequential: 順序カラーパレット

  • 明度や彩度が連続的に変化

  • 一方向に増加する数値を表現するときに利用する

Viridis: 数値の変化と視覚的変化が一致するように数学的に設計

Diverging: 発散カラーパレット

  • 基準色を中心に両極端へ向けて変化する配色

  • ゼロや平均値などを基準に持つ変数の可視化に利用する

Temps: 色彩豊かな発散パレット

Storytelling With Data

  • 探索的な可視化

    • データの識別性を高めるために審美的属性を利用する

  • 説明的な可視化: 伝えたいメッセージを強調

    • 注目すべきデータを強調するために審美的属性を利用する

    • 主役以外の部分をグレーアウトし、読者の視線を誘導する
